What are UX and UI?





UX = User Experience

Abbreviated to UX Design, User Experience Design is a practice that is involved with laying out and adjusting web pages and their elements in a way that is optimal for the user visiting the page.

This includes thinking about what words to use, what labels to have, icons vs text, placement, ordering, spacing, and more! Having a good UX design will make users enjoy the site more and make it easier for them to use the site.




UI = User Interface

Abbreviated to UI Design, User Interface Design is a practive that is involved with designing aesthetically-pleasing pages and graphics. To achieve a successful UI design for your websites, use the elements of art (line, shape, color, value, form, texture, and space).

While using these elements, keep in mind the principles of art: balance, contrast, emphasis, movement, pattern, rhythm, and unity/variety.




How can we implement good UX / UI?


To implement good UX / UI, we can follow some curated tips from professionals. These will be helpful for us in starting to think critically about the componenent of strategy that goes into designing products and websites!





Don't use blue as a text color other than for links.

Buttons must look clickable and have enough space for users to click or tap comfortably!

Look at color psychology when choosing colors.

Navigation should not change throughout the site.

Remember that every element carries a visual weight. With this in mind, strive for either symmetrical or asymmetrical balance in your designs.

Repetition in design (through colors, shapes, fonts, etc.) reinforces ideas.

Rhythm in design can be random, regular, alternating, flowing, or progressive.

Be mindful of movement = the way your eye travels over a design.

Variety creates visual interest -- the more interesting and aesthetically-pleasing a design is can positively impact the user's perception of the site.

The home page should include the site identity and mission, information about what can be found on the site, teases, and an idea where users should start.

Use as much space as necessary but not any more than that.

Use heading tags correctly. H1 is for a main content header, H2 marks major sections, H3 is for subheadings, etc.

Make sure there is enough contrast between text and its background.

Don't float headings between paragraphs.

Standardize and ensure you explain any abbreviations.

Links should look like links.

Choose up to three font families.

Choose up to four different font sizes.

Group and sequence menu items meaningfully.

Put key information toward the top of the page and first.

Avoid presenting information in carousels. Each consecutive slide erases the memory of the information presented on the previous one -- users can only focus on one thing.

Navigation labels should be no more than 2-3 words, specific, and start with the most information-carrying word.

Use accordians to compress lengthy content on mobile sites.

The main purpose of each webpage should be obvious to a user.

Use color and size contrast to differentiate primary and secondary info.

Make icons simple, familiar, and meaningful.

Put time into pairing your fonts and thinking about how they work together.

Respect the space of other elements. Give elements space to breathe with sufficient spacing.

For color schemes, aim for 1-3 primary colors and 1-3 secondary colors that constrast and complement each other.

If you are placing text on background images, make sure to lower the brightness of the images to make the text easier to read.

Keep it simple, but don't forget the necessities.

Remember that fonts represent ideas. Rounded edges are considered to be perceived as friendlier, hard edges imply strength, and serifs (like Times New Roman) are considered elegant and sophisticated.

Simple, obvious terms are better for navigation than industry jargon or trendy terms.

If information is presented in a table, focus on formatting.

Do not make main content text too small -- use at least 11 pt for size.

When presenting data, ensure it is done in a way that is not misleading or hard to draw meaning from.

Only put elements next to each other if they are exactly the same size or considerably different sizes.

Don't use two different serif fonts next to each other or two different italicized fonts next to each other; fonts should have considerably different styles.

Consider legibility, readability, weights, and styles when choosing fonts.

Italicized and uppercase text can be harder to read, especially for dyslexic users.

Site logo should generally go in the top left corner, be medium size, and not be crowded by other elements.

Define the alt attribute for img tags!

Add a favicon! It helps brand your site further and is used when a site is bookmarked in a browser.

Create order using alignment, such as by using a line alongside text.

Don't use gray text on colored backgrounds.

Use grids or frames to align images.

Have clear and concise headlines.

Use accent partial-borders to add color to a bland design.

Not every button needs a filled in background -- just use an outline. Filled in backgrounds should be reserved for primary buttons, and buttons with just outlines should be used for secondary buttons.

We don't necessarily read pages, we scan them. Format content to support this!

Conventions (how things look, where they are, how they work) are your friend -- take advantage of them.

Break up pages into clearly defined areas.

Eliminate distractions.

Don't make users have to think!

Break up long text into sections with subheadings.

Innovate when you know you have a good idea, take advantage of conventions when you do not.

Elements related logically also need to be related visually by being in the same area, being grouped, or being styled the same.

Omit needless words. Doing so will reduce the noise of the page, accent the useful information, and make your pages shorter and thus easier to scan.

Keep paragraphs short.

Use "You are Here" indicators in your navigation.

Do not use a corporate mission statement as a welcome blurb.

Use bullet points when available.

Don't go overboard with fancy fonts. Use a small number of elegant fonts consistently instead.

Make sure calls-to-action are highly visible and don't require users to search for them.

Include a title and description in your site's metadata (in the head tag at the top of your HTML documents).

Try using a box shadow vs a border on some elements!

Don't overuse borders.

Try not making icons bigger than they were designed to be. Instead, try giving them added size by placing them on top of a shape, like a circle!

Strive for consistency, but remember clarity DOES trump consistency.

Use consistency for colors, grids/alignments, styles, and fonts.

Use as few decorative elements as possible -- only use the essentials to reduce visual noise. This means not using any unnecessary shadows, frames, icons, patterns, etc.

Whatever doesn't spark joy has to go.

Make sure labels are next to the element they describe.
